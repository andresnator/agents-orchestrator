# ADR Markdown Template

Use this structure for the final ADR. Replace placeholders with user-provided or explicitly confirmed content.

```markdown
# [Title]

## Status

[Proposed | Accepted]

## Context

[Describe the problem, constraints, forces, and timing that require a decision. Focus on why a choice is needed, not only on the selected solution.]

## Decision

[State the selected or proposed option and the rationale for choosing it.]

## Options Considered

### Option 1: [Name]

[Brief description.]

**Pros:**
- [Advantage 1]
- [Advantage 2]

**Cons:**
- [Drawback 1]
- [Drawback 2]

### Option 2: [Name]

[Brief description.]

**Pros:**
- [Advantage 1]

**Cons:**
- [Drawback 1]

## Consequences

**Positive:**
- [Positive consequence]

**Negative:**
- [Negative consequence or trade-off]

## Follow-up

- [Open question or action, only when useful]
```

## Notes

- Omit `Follow-up` when there are no useful open actions or unresolved questions.
- Add or remove option sections to match the alternatives the user actually provided or confirmed.
- Do not invent alternatives, pros, cons, or rationale.
