# Spec Question Bank

Ask one at a time, in the user's language, with a recommended answer.

1. Which capability owns this behavior, and is it new or modified?
2. What MUST the system do from the user's or external system's view?
3. What trigger/action starts the behavior?
4. What observable result proves the behavior worked?
5. What edge case, invalid input, permission, or failure state must be specified?
6. Which existing requirement is replaced, removed, or renamed?
7. For MODIFIED behavior, what full requirement should remain after archive?
8. What design/implementation note should be parked outside the spec?
