# Design Question Bank

Ask one at a time only after read-only code exploration cannot answer it.

1. Which existing pattern should this change extend, and where is the best seam?
2. Which files are modified versus created, and why?
3. What interfaces, commands, routes, events, or data contracts change?
4. What state/data flow proves the design is coherent?
5. What alternative approach is viable, and why is it rejected?
6. What migration, rollout, compatibility, or rollback concern exists?
7. What tests should prove each spec scenario and key integration?
8. What unknown should remain an Open Question instead of being guessed?
